Console

You can select the `Via console` trigger operation while
`creating an event trigger <create_trigger>` to allow invoking the event
trigger on rows manually using the Hasura console *(available after
version v1.0.0-beta.1)*.

In the `Data -> [table-name] -> Browse Rows` tab, clicking the
`invoke trigger` button next to any row lets you invoke "manual event
triggers" configured on the table with that row as payload *(the button
will be shown only if you have any triggers configured)*:

<div class="thumbnail" alt="Invoke event trigger on console">

/img/graphql/core/event-triggers/select-manual-trigger.png

</div>

Click on the event trigger you want to run and a modal will pop up with
the request and response.

<div class="thumbnail" alt="Request and response of event trigger">

/img/graphql/core/event-triggers/run-manual-trigger.png

</div>

<div class="tab">

API

When creating an event trigger over the
`create_event_trigger metadata API<create_event_trigger>`, you can set
the argument `enable_manual` to true.

Then you can use the
`invoke_event_trigger metadata API<invoke_event_trigger>` to invoke
triggers manually:

```http
POST /v1/query HTTP/1.1
Content-Type: application/json
X-Hasura-Role: admin

{
   "type" : "invoke_event_trigger",
   "args" : {
      "name": "send_email",
      "payload": {}
   }
}
```
