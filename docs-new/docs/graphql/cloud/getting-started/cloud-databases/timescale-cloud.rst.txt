# Using Hasura Cloud with a Timescale Postgres database

<div class="contents" backlinks="none" depth="2" local="">

Table of contents

</div>

## Introduction

This guide explains how to connect a new or existing Timescale Postgres
database to a Hasura Cloud project.

## Step 0: Sign up or log in to Hasura Cloud

Navigate to [Hasura
Cloud](https://cloud.hasura.io/signup/?pg=docs&plcmt=body&cta=navigate-to-hasura-cloud&tech=default)
and sign up or log in.

## Step 1: Create a Hasura Cloud project

On the Hasura Cloud dashboard, create a new project:

<div class="thumbnail" alt="Create Hasura Cloud project" width="1000px">

/img/graphql/cloud/cloud-dbs/create-hasura-cloud-project.png

</div>

You will get prompted for a Postgres Database URL. We will create this
in the next step and then come back here.

After the project is initialized successfully, click on `Launch console`
to open the Hasura console in your browser.

On the Hasura console, navigate to
`Data -> Manage -> Connect Database -> Connect existing database`:

<div class="thumbnail" alt="Hasura Cloud database setup" width="700px">

/img/graphql/cloud/cloud-dbs/existing-db-setup.png

</div>

## Step 2: Create a Postgres DB on Timescale (skip if you have an existing DB)

Log into the [Timescale Cloud
portal](https://portal.timescale.cloud/login).

On the Timescale dashboard, under `Services`, click on
`+ Create new service`:

<div class="thumbnail" alt="Create a new service on Timescale"
width="1000px">

/img/graphql/cloud/cloud-dbs/timescale/create-new-service.png

</div>

Select the Postgres option:

<div class="thumbnail" alt="Select Postgres on Timescale" width="700px">

/img/graphql/cloud/cloud-dbs/timescale/select-postgres.png

</div>

Scroll down and select the `Cloud Provider`, `Region` and `Service Plan`
based on your requirements.

In the end, enter a `Name` for the service:

<div class="thumbnail" alt="Create a service on Timescale"
width="1000px">

/img/graphql/cloud/cloud-dbs/timescale/create-service.png

</div>

Then click `Create service`.

<div class="note">

<div class="title">

Note

</div>

If you're using a database user other than the default one, make sure to
give it the right `Postgres permissions <cloud_postgres_permissions>`.

</div>

## Step 3: Allow connections to your DB from Hasura Cloud

On the `Services` dashboard, click on your DB:

<div class="thumbnail" alt="Select DB on Timescale" width="1000px">

/img/graphql/cloud/cloud-dbs/timescale/select-db.png

</div>

Scroll down to `Allowed IP Addresses` and click on `Change`:

<div class="thumbnail" alt="Change allowed IP addresses on Timescale"
width="1000px">

/img/graphql/cloud/cloud-dbs/timescale/change-allowed-ip-addresses.png

</div>

Copy the IP address from the copy icon in the `Hasura Cloud IP` field on
the project's details view on Hasura Cloud.

<div class="thumbnail" alt="Hasura Cloud IP field" width="1000px">

/img/graphql/cloud/projects/hasura-cloud-ip.png

</div>

Add the Hasura IP address that you copied, click on the `+` and then
click on `Save changes`:

<div class="thumbnail" alt="Add the Hasura IP on Timescale"
width="500px">

/img/graphql/cloud/cloud-dbs/timescale/add-hasura-ip.png

</div>

## Step 4: Get the database connection URL

The structure of the database connection URL looks as follows:

```bash
postgresql://<user-name>:<password>@<public-ip>:<postgres-port>/<db>
```

To get it, navigate to the `Overview` tab of your database dashboard and
copy the `Service URI`:

<div class="thumbnail" alt="Copy the service URI on Timescale"
width="1000px">

/img/graphql/cloud/cloud-dbs/timescale/copy-service-uri.png

</div>

## Step 5: Finish connecting the database

Back on Hasura Console, enter the database URL that we retrieved in
`step 4 <get_db_url_timescale>`:

<div class="thumbnail" alt="Database setup" width="600px">

/img/graphql/cloud/getting-started/connect-db.png

</div>

Then click `Connect Database`.

<div class="note">

<div class="title">

Note

</div>

For security reasons, it is recommended to set database URLs as
`env vars <manage_project_env_vars>` and using the env vars to connect
to the databases in place of the raw database URLs.

</div>

Voil√†. You are ready to start developing.

<div class="thumbnail" alt="Hasura console" width="1100px">

/img/graphql/cloud/cloud-dbs/hasura-console.png

</div>
